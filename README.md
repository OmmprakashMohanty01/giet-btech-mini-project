# Contributing Guidelines

## How to Contribute

If you'd like to contribute, please fork the repository and make changes to the desired file(s). Don't forget to update the CHANGELOG (if applicable) and run the tests to ensure everything still works as expected.

## Commit Messages

- Use the present tense (e.g. 'Add' instead of 'Added')
- Be descriptive but concise (maximum 72 characters)
- Use bullet points for multiple changes

## Pull Request Format

- Provide a clear and descriptive title
- Explain what you did in the description
- Link to the relevant issue (if any)

## Code Standards

- Follow PEP 8 for Python code
- Write tests for new features
- Update any affected documentation (if applicable)

## Citing this Repository

If you use this repository as a reference in your work, please include a citation in your document (e.g. paper, thesis, or blog post).
